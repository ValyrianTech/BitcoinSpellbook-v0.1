API documentation
Blocklinker: LinkedList

Returns an ordered list of address-address or address-value pairs where the first addresses come from a SIL and are linked to the corresponding address in a HD-wallet.

The first addresses are the PrimeInputAddresses of all transactions sent to this address, ordered first by blockheight, then transactionhash. For "Linked Address List", the second addresses are the addresses generated by the extended public key (XPUB). For the other 3, the values are the corresponding balance, total received or total sent.

Arguments:
Name Required Type Description
format yes string Format of the results, should be “json”
address yes string A Bitcoin address
block no integer A block height, if omitted the latest blockheight is used.
XPUB yes string An extended public key (BIP32)
metric yes string "LAL", "LBL", "LRL" or "LSL". Multiple metrics seperated with ","
Example: LinkedList?format=json&address=1NC8LqAB99bYM9wVoD2grdYMMZAhjwy57A&xpub=xpub6CUvzHsNLcxthhGJesNDPSh2gicdHLPAAeyucP2KW1vBKEMxvDWCYRJZzM4g7mNiQ4Zb9nG4y25884SnYAr1P674yQipYLU8pP5z8AmahmD&metric=LAL,LBL,LRL,LSL&block=376000
Return values:
Name Type Description
success integer 1 = success, 0 = fail
LAL array ordered list of address-address pairs
LBL array ordered list of address-value pairs
LRL array ordered list of address-value pairs
LSL array ordered list of address-value pairs
error string error message in case something went wrong
